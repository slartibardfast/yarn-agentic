## Workload definitions (locked at T5.9.C)

### M3-steady (regression) — production-comparable

- `bash scripts/bench-t3.8-m3.sh` (NP=8, ctx_per_slot=8192, Hadamard
  K/V on, `--no-context-shift`, locked clocks).
- Same workload used for T5.4 GP5.a Bundle A regression close
  (26.50 t/s mean, CV 0.14%).
- **GP5.9.regression band:** ≥ T5.8 baseline 26.45 × 0.98 = 25.92 t/s.

### M4-bigctx (feasibility) — paged-BACKING capability target

- `bash scripts/bench-ctx-feasibility.sh` (sibling profile
  `qwen36-27b-x2-dflash-bigctx.sh`, ctx_per_slot=1048576, NP=8,
  `--kv-pool-blocks 16384` = 1M physical-token capacity, oversubscribed
  by N_PROMPTS=16 > NP=8).
- Wall-time measurement is secondary; PRIMARY gate is the
  triple-conjunction:
    1. ≥ 1 request returns 200 with non-empty completion (feasibility).
    2. ≥ 1 request returns 503 + Retry-After (admission gate works).
    3. `/health` returns 200 after burst (server stays up).
- This workload **could not run pre-T5.9** — contiguous KV would need
  ~320 GiB (1M tok × 8 NP × 20 KB/token Q4_0 × 2 layers) vs 48 GiB
  available, and `llama_kv_cache_init` would fail allocation before
  any inference. Paged BACKING makes ctx-per-slot decouple from
  physical-pool sizing.

### Cross-NP determinism (NPC) — invariance

- `bash scripts/verify-production-determinism.sh
   NP_LIST="1 2 4 8" CTX_CHECKPOINTS=3` (already T5.8 baseline).
- **GP5.9.NPC:** ACCEPTANCE PASS at production sizes.
- **GP5.9.batch-shape invariance:** all 4 tests
  (`libllama-verify-batch-width-sweep`,
   `libllama-multi-cycle-restore-drift`,
   `kernel-mulmat-batch-shape`,
   `kernel-mulmat-mmq_x-dispatch`) PASS.

## VRAM budget at the M4-bigctx workload

At NP=8, ctx_per_slot=1048576:

| Component | Contig (would need) | Paged BACKING (this profile) |
|---|---:|---:|
| Main model weights | 13 GiB | 13 GiB |
| Main KV (Q4_0, layers × heads × ctx × NP) | ~320 GiB | sized by `--kv-pool-blocks 16384` = 16384 × 64 = 1 048 576 physical tokens — ~20 GiB |
| Compute scratch | 3 GiB | 3 GiB |
| **Total** | **~336 GiB** ❌ does not fit 48 GiB | **~37 GiB** ✓ fits with headroom |

Decoupling: `--ctx-size 8 388 608` × `--parallel 8` defines the
nominal_max; `--kv-pool-blocks 16384` defines the physical pool;
admission gate emits 503+Retry-After when the union of in-flight
slots' demand would exceed the pool.

This is the user-override-defined T5.9 deliverable per
`[[project_t5_probe_falsified_path_c_override]]`: forward-looking
infra for ctx ≥ 1M workloads where contig can't allocate.
